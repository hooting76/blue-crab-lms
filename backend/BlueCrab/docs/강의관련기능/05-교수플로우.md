## 1. 강의 사전 준비 플로우

### **1.1 강의 목록 조회 및 관리**

```mermaid
sequenceDiagram
    participant Professor as 교수
    participant UI as 교수 UI
    participant API as 강의 API
    participant DB as 데이터베이스

    Professor->>UI: 강의 관리 메뉴 선택
    UI->>API: GET /api/professor/lectures?year=2025&semester=1
    API->>API: 교수 권한 검증
    API->>DB: 담당 강의 목록 조회
    DB-->>API: 강의 목록 데이터
    API-->>UI: 강의 목록 표시

    Professor->>UI: 강의 선택
    UI->>API: GET /api/professor/lectures/{id}
    API->>DB: 강의 상세 정보 조회
    DB-->>API: 강의 상세 데이터
    API-->>UI: 강의 상세 정보 표시
```

### **1.2 강의 정보 수정**

```mermaid
sequenceDiagram
    participant Professor as 교수

    participant API as 강의 API
    participant DB as 데이터베이스

    UI->>UI: 강의 정보 수정 폼 표시
```

## 3. 출결 관리 플로우

### **3.1 출석 현황 조회**

```mermaid
sequenceDiagram
    participant Professor as 교수
    participant UI as 교수 UI
    participant API as 출결 API
    participant DB as 데이터베이스

    Professor->>UI: 출결 관리 선택
    UI->>API: GET /api/professor/lectures/{id}/attendance
    API->>DB: 출석 데이터 조회
    Note over DB: 날짜별 출석 현황,<br/>학생별 출석률 계산

    DB-->>API: 출석 데이터
    API-->>UI: 출석 현황 표시
    UI-->>Professor: 출석 현황 확인

    Professor->>UI: 특정 날짜 선택
    UI->>API: GET /api/professor/attendance?date=2025-03-15
    API->>DB: 해당 날짜 출석 조회
    DB-->>API: 날짜별 출석 데이터
    API-->>UI: 날짜별 출석 상세 표시
```

### **3.2 출석 승인 처리**

```mermaid
sequenceDiagram
    participant Professor as 교수
    participant UI as 교수 UI
    participant API as 출결 API
    participant DB as 데이터베이스
    participant Student as 학생 시스템

    Professor->>UI: 출석 요청 목록 확인
    UI->>API: GET /api/professor/attendance/requests
    API->>DB: 미승인 출석 요청 조회
    DB-->>API: 출석 요청 목록

    Professor->>UI: 출석 승인/거부 선택
    UI->>API: PUT /api/professor/attendance/{id}/approve
    API->>API: 승인 처리
    API->>DB: 출석 상태 업데이트
    DB-->>API: 업데이트 완료

    API->>Student: 출석 승인 알림 발송
    Student-->>API: 알림 확인

    API-->>UI: 승인 완료
    UI-->>Professor: 출석 승인 처리 완료
```

## 📋 교수 주요 업무 프로세스

### **학기 전 준비**
1. **강의 계획 검토**: 배정된 강의 확인 및 준비
2. **강의 정보 보완**: 설명, 목표, 평가방법 작성
3. **공지사항 작성**: 수강신청 전 주요 공지 등록

### **수강신청 기간**
1. **수강생 모니터링**: 등록 학생 현황 확인
2. **문의 대응**: 학생 질문에 채팅으로 응대
3. **강의 준비**: 세부 커리큘럼 및 자료 준비

### **학기 중 운영**
1. **출결 관리**: 매주 출석 승인 및 모니터링
2. **강의 진행**: 공지사항 등록, 자료 공유
3. **과제 관리**: 과제 출제 및 채점
4. **학생 상담**: 개별 학생과의 채팅 상담

### **학기 말 정리**
1. **성적 입력**: 모든 평가 요소 입력 및 확정
2. **피드백 수렴**: 학생들의 강의 평가 확인
3. **다음 학기 준비**: 개선사항 정리 및 계획
